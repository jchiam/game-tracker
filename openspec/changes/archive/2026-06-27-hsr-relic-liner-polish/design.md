## Context

HSR `CharacterCard` currently renders a `.game-card-static-line` in the collapsed summary showing relic set names + counts. Full names are long (3-5 words), causing ellipsis truncation on narrow cards. The shared `.game-card-static-line` in `card.css` has `white-space: nowrap; text-overflow: ellipsis`. The collapsed height varies between cards with/without relics since the one-liner is absent when no relics are equipped.

`relic_sets.ts` is auto-generated by `scripts/update-hsr-data.mjs` and must not be hand-edited.

## Goals / Non-Goals

**Goals:**

- Abbreviated names for all ~58 relic sets (hand-maintained mapping).
- 2-line wrapping for the one-liner to fit mixed sets without truncation.
- Fixed 2-line height reservation for uniform collapsed height across all cards.

**Non-Goals:**

- Not modifying the auto-generated `relic_sets.ts` file.
- Not modifying the update script.
- Not changing other games' one-liner behavior.
- Not adding short names to the `RelicSet` interface (kept separate to avoid coupling data generation with display concerns).

## Decisions

**Decision: Separate `RELIC_SHORT_NAMES` map file, not in the interface.**
Create `src/data/honkai-star-rail/relic_short_names.ts` exporting a `Record<string, string>` mapping relic set IDs to short display names. This avoids touching the auto-generated `relic_sets.ts` and keeps the `RelicSet` interface unchanged. The component does `RELIC_SHORT_NAMES[setId] ?? fullName` as fallback.

**Decision: Short names are the most distinctive word(s) from each set name.**
Examples:

- "Champion of Streetwise Boxing" → "Streetwise"
- "Band of Sizzling Thunder" → "Thunder"
- "Firesmith of Lava-Forging" → "Firesmith"
- "Broken Keel" → "Keel"
- "Musketeer of Wild Wheat" → "Musketeer"
- "Sacerdos" → "Sacerdos" (already short)

Goal: each short name is unique across all sets, recognizable to players, and ≤12 chars where possible.

**Decision: HSR-local CSS override for 1-line height reservation.**
Override `.game-card-static-line` in `CharacterCard.css`:

```css
.game-card-static-line {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
  min-height: calc(var(--typography-line-height-base) * 1em);
}
```

Single line with ellipsis truncation for long names. The `min-height` reserves 1 line even when showing the dash (no relics equipped), ensuring uniform collapsed card height.

**Decision: Summary height budget stays 100px.**
One line of text ≈ 14-16px. Chips row ≈ 32px. Padding ≈ 20px. Total ≈ 66-68px — fits within 100px.

## Risks / Trade-offs

- **[Maintenance of short names]** New relic sets added by the update script won't have short names until manually added. Fallback to full name prevents breakage — just looks long until the mapping is updated.
- **[1-line height reservation with dash]** When no relics equipped, the dash sits in a 1-line-tall area. Minimal wasted space while preserving uniform card height.
