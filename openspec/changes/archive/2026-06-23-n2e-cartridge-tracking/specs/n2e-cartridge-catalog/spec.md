## ADDED Requirements

### Requirement: Cartridge catalog data file

The system SHALL maintain a static catalog of named N2E cartridges in `src/data/neverness-to-everness/cartridges.ts`, auto-generated by `scripts/update-n2e-data.mjs`. The catalog SHALL NOT be edited manually.

Each entry SHALL have the fields: `id` (string, format `{base}_{quality}`), `name` (string, human-readable set name), `rarity` (string, one of `'B' | 'A' | 'S'`), and `imageUrl` (string, reserved for future use, set to empty string).

#### Scenario: Catalog generated from API

- **WHEN** the update script runs and the `shards` GraphQL query returns items with `type_id === 'core'`
- **THEN** `cartridges.ts` is written with all core entries grouped by rarity (S first, then A, then B), sorted alphabetically by name within each group

#### Scenario: New cartridge set added in game patch

- **WHEN** a new core entry appears in the `shards` query response on the next weekly run
- **THEN** the diff report logs `+1 added` and the new entry is included in the regenerated file

#### Scenario: Catalog accessible at runtime

- **WHEN** any component imports `ALL_CARTRIDGES` from `@/data/neverness-to-everness/cartridges`
- **THEN** the full array of `N2ECartridge` entries is available with correct `id`, `name`, and `rarity` fields

### Requirement: Cartridge catalog update script integration

The system SHALL extend `scripts/update-n2e-data.mjs` to fetch, process, and generate `cartridges.ts` as part of the existing weekly run. No separate workflow or flag is required.

#### Scenario: Script runs without ImageKit credentials

- **WHEN** `IMAGEKIT_PRIVATE_KEY` is not set and the script runs
- **THEN** `cartridges.ts` is still generated correctly (no image assets are involved)

#### Scenario: Diff report includes cartridge changes

- **WHEN** the script completes a run
- **THEN** the console output includes a cartridges summary line showing total count and added/removed entries since the last run
